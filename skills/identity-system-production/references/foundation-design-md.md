# Foundation Design Markdown

Use `BRAND_FOUNDATION_DESIGN.md` as the AI-readable foundation document for the brand.

This is not a replacement for the Brand Identity Guideline. It is the design-facing version of the system, written so downstream design agents and tools can generate work consistently.

## Why It Exists

Traditional guidelines are often written for people. AI tools need dense, explicit, prompt-friendly structure. A good foundation design file removes ambiguity from later generation work.

## Required Sections

1. Visual Theme and Atmosphere
2. Brand Intent and Perception
3. Color Palette and Roles
4. Typography Rules
5. Logo System and Lockups
6. Graphic System and Shapes
7. Imagery and Art Direction
8. Layout Principles
9. Motion and Interaction Cues when relevant
10. Dos and Don'ts
11. Agent Prompt Guide

## Writing Rules

- Prefer direct, observable rules over abstract adjectives.
- Include actual values whenever they exist.
- Explain behavior, not only inventory.
- State what must stay stable.
- State where adaptation is allowed.

## Downstream Usage

This file should be usable by:

- Stitch
- Figma MCP
- Pencil or similar design agents
- future wrapper skills or automation layers
