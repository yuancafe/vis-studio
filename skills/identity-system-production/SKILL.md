# Identity System Production

## Overview

Handle the convergent refinement phase. Turn the selected concept into a real, technically robust identity system that works across digital and physical touchpoints. Stop at core-system quality and foundational application proof, not at mass collateral production.

## Why This Skill Exists

NotebookLM synthesis separates concept exploration from final production. This phase is about engineering: responsive logo logic, production specs, typography discipline, foundational application testing, system coherence, and preparing the first stable `brand-pack` foundation.

## Entry Requirement

Begin only after concept approval from `visual-concept-exploration`.

## Workflow

### 1. Refine the mark system

Define:

- primary logo
- secondary lockups
- icon / favicon / avatar
- monochrome behavior
- clear space
- minimum size

### 2. Lock the color system

Define:

- primary colors
- support colors
- accent colors
- neutrals
- role logic
- values across required formats

### 3. Lock the typography system

Define:

- family count
- hierarchy
- weights
- spacing rhythm
- fallback and licensing logic

### 4. Build supporting system elements

Define:

- imagery
- photography direction
- iconography
- patterns or shapes
- layout behavior

### 5. Test on real touchpoints

Preview on:

- web
- social
- deck
- print / packaging if relevant

Keep these as proof points and foundational application rules, not full production batches.

### 6. Draft the AI-readable foundation design file

Create `BRAND_FOUNDATION_DESIGN.md` as an AI-facing design system document inspired by Stitch-style `DESIGN.md` structure, but expanded for full VIS work.

It should capture:

- visual theme and atmosphere
- brand intent and perception
- color palette and semantic roles
- typography rules and hierarchy
- logo system behavior
- graphic system and shapes
- imagery and art direction
- photography direction and lighting rules
- layout principles
- motion or interaction cues when relevant
- dos and don'ts
- agent prompt guide for downstream tooling

This file should be specific enough that tools such as Stitch, Figma MCP, or similar design agents can generate work that stays on-system.

### 7. Stop for the 75% gate

Secure approval on the high-fidelity system before governance packaging.

### 8. Draft the brand-pack foundation

Create the machine-readable base for downstream automation:

- `brand-guideline`
- `brand-tokens`
- `logo-system`
- `style-playbook-selection`
- `design_md_documents.brand_foundation_design`

## Outputs

- production-ready logo system
- color specs
- typography hierarchy
- imagery and graphic rules
- photography direction
- foundational application proofs
- `BRAND_FOUNDATION_DESIGN.md`
- draft brand-pack foundation

## Non-Negotiables

- Start logo judgment in black and white
- Keep typography disciplined, usually 2-3 families
- Specify colors in all required output systems
- Test small-scale, digital, and print behavior where relevant
- Do not treat one polished mockup as proof of a scalable system
- Do not let this phase expand into a giant application list

## Hand Off

After approval, continue with `brand-application-system`.
