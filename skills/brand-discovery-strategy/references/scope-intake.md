# Scope Intake

Collect scope signals early so downstream application work is not guessed later.

Ask:

- Is this engagement strategy-only, identity-only, or identity plus applications?
- Which touchpoints matter in the first release?
- How many applications are expected now versus later?
- Which items are business-critical versus nice-to-have?
- Which items are likely templated versus custom-crafted?

Record any answer that changes:

- workload
- pricing posture
- phase boundaries
- likely production routes
