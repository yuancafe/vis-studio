# Application Design Markdown Outputs

This phase should produce design-facing markdown documents that extend the brand foundation into real touchpoints.

## File Set

### `UI_UX_DESIGN.md`

Create this when digital products or marketing surfaces are in scope:

- website
- landing page
- dashboard
- product UI
- signup flow
- app screens

Include:

- page atmosphere
- navigation and page structure
- section patterns
- card and content modules
- forms, inputs, CTAs, and states
- responsive behavior
- motion and interaction cues
- prompt-ready component instructions

### `APPLICATION_DESIGN.md`

Create this for broader application work:

- social templates
- decks
- print
- packaging
- signage
- email headers
- banners
- event graphics

Include:

- touchpoint families
- layout rules by family
- size behavior
- export behavior
- route logic
- prompt-ready application instructions

## Writing Rules

- Extend the foundation document rather than restating it.
- Keep each family actionable.
- Use family-specific constraints.
- State what is template-safe versus what needs manual craft.
