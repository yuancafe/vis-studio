# Application System Workflow

## 1. Normalize the request

Turn raw asks into a structured application list.

## 2. Prioritize

Separate must-have, should-have, and later items.

## 3. Classify the route

For each item choose:

- `template_auto`
- `mockup_semi_auto`
- `manual_vector`

## 4. Produce route-specific outputs

- template outputs for scalable collateral
- mockup outputs for presentation-led applications
- manual briefs for vector-heavy work

## 5. Capture recipes

Store enough logic that the item family can be regenerated later.
