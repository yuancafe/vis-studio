# Reference Style Distillation

Use this workflow when the team has collected many reference brands, sites, campaigns, or identity systems and needs to turn them into original direction.

## Goal

Produce `REFERENCE_STYLE_DISTILLATION.md` as a synthesis document, not a scrapbook.

The document should help later phases answer:

- what visual signals recur across references
- which signals are strategically useful
- which signals should be adapted rather than copied
- which signals should be rejected
- how the chosen signals translate into our own identity system

## Input Range

Use 8-15 references when possible. Fewer than 5 usually produces weak pattern recognition. More than 15 often becomes noisy unless the set is clustered first.

## Extraction Grid

For each reference, extract:

- reference name and category
- what makes it relevant
- atmosphere and posture
- typography behavior
- color behavior
- logo posture
- layout and composition behavior
- imagery or material cues
- component or application cues
- what to keep
- what to adapt
- what to avoid

## Synthesis Rules

- Do not average everything into a generic middle.
- Look for repeated structural signals, not only aesthetic similarities.
- Separate category convention from distinctive edge.
- Mark any copied risk explicitly.
- Convert observations into original design rules.

## Required Output Sections

`REFERENCE_STYLE_DISTILLATION.md` should include:

1. Reference roster
2. Repeating signals
3. Strategic fit signals
4. Keep / Adapt / Avoid rules
5. Territory hypotheses
6. Translation notes for logo, color, type, imagery, and applications
7. Anti-copy guardrails

## Use With Playbooks

After the distillation, compare the result with the structured playbook library. The references should sharpen the playbook choice, not override strategy.
