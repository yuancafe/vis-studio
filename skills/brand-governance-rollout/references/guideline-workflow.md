# Guideline Workflow

## 1. Document only approved rules

Do not turn unresolved experiments into official standards.

## 2. Explain the why

For each rule, document:

- what the rule is
- why it exists
- how to apply it
- what misuse looks like

## 3. Package for use

The guideline should support real execution, not only presentation.

Include:

- export formats
- access instructions
- naming conventions
- update logic

## 4. Prioritize near-term rollout

Build for the touchpoints that matter now. Do not overproduce low-value templates.
