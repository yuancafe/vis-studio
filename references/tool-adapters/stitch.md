# Stitch Adapter

Use this adapter when generating or refining digital UI through Stitch.

## Priorities

- product surface tone
- section patterns
- UI hierarchy
- responsive logic
- prompt-ready component behavior

## Read first

- `BRAND_FOUNDATION_DESIGN.md`
- `UI_UX_DESIGN.md`

## Emphasize

- page goals and user surface
- layout modules
- navigation patterns
- buttons, cards, forms, and states
- brand-specific dos and don'ts

## Avoid

- print-first language
- abstract brand copy without concrete UI instructions
- letting Stitch guess the interaction tone
