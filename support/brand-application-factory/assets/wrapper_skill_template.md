# {brand_name} Brand Applications

## Overview

Use the bundled brand pack at `{pack_rel_path}` as the single source of truth for post-system application work.

## Supported intents

### 1. List bundles

Read `{bundle_ref_rel_path}` and summarize which application families and sample bundles are available.

### 2. Restate constraints

Read `assets/brand_pack.json` and restate:

- logo-system limits
- typography rules
- color architecture
- approved playbook posture
- application recipe boundaries

### 3. Generate automation-friendly applications

Use the bundled pack to describe or prepare work for:

- template-friendly applications
- scalable touchpoints
- repeated size variants

### 4. Produce manual briefs

When the requested item is `manual_vector`, do not fake automation.

Produce a clean execution brief that includes:

- application goal
- critical brand constraints
- route reason
- export expectations
- handoff notes for manual vector production

## Operating rules

- Prefer the bundled pack over memory or guesswork
- Keep new application work inside the approved brand system
- Flag any request that requires changing the underlying identity system
